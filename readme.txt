后台管理二次开发说明

一、命名规范
1.后台ajax获取内容中的函数规范
必须使用function funcname(){} 格式，否则可能会无法加载函数

2.防止命名冲突
由于后台加载页面未采用iframe方式，因此要防止id和函数名等重复；
以下规范仅供参考：
(a)HTML默认id为下划线连接格式，如：controllername_actionname_other
(b)js中变量命名同id一致
(c)js函数名为首字母小写的驼峰命名，如controllernameActionnameOther

3.其它规范请参考ThinkPHP官方手册

二、后台菜单规则
左侧显示方式为：
一级菜单
├─┬ 二级菜单
│ ├─ 三级菜单
│ └─ 三级菜单
└─┬ 二级菜单
  └─ 三级菜单

1.一级菜单（顶部菜单）
2.二级菜单（左侧分类菜单）
3.三级菜单（左侧可点击菜单）
4.四级菜单（不做显示用，用于权限控制）

三、开发手册
因为本系统都是采用的开源框架，因此如果有不清楚的可以参考对应手册
1.PHP请参考ThinkPHP3.2官方手册
2.easyui请参考easyui官方手册
